# Commands

Replace this file with a compact, polished command reference for the generated
tool.

Keep it suitable for another agent to consume quickly:

- how to invoke the CLI locally
- how to invoke the published CLI, if it is published to JSR
- command name
- required flags
- optional flags
- stable identifiers or output fields it emits
- one-line purpose

If the tool uses browser-auth session reuse, show `--session-path` as a required
flag on `login` and every later authenticated command.

If commands compose with each other, show which identifier or output field each
later command consumes.

Keep this file aligned with the command names and help text generated by
`@cliffy/command`.

Keep it terse and operational. Do not repeat long explanations that already
belong in `README.md`.

If the package is intended to be published to JSR, also show the published
invocation form:

- `deno x jsr:@scope/package/cli`
- `deno install -g -A jsr:@scope/package/cli`
